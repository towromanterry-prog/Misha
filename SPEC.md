# SPEC

## Требования

- Telegram удаляем полностью: интеграции, кнопки, обработчики и любые сценарии отправки через Telegram.
- **ONLINE**: отчёт отправляется по email через SMTP.
- **OFFLINE**: используется сценарий `QR -> mailto`.
- В офлайн-письме должен быть только результат `tracert -d 77.88.8.8` по первым 5 хопам.
- В OFFLINE-отчёте не должно быть имени ПК (`hostname`).

## UI (расположение кнопок)

- Кнопка **«Отправить email»** расположена в основном блоке отправки отчёта (главное действие).
- Кнопка **«QR (если нет интернета)»** расположена рядом с кнопкой «Отправить email» как альтернативный сценарий для OFFLINE.

## Точки вызова (entry points)

> На текущем шаге в репозитории отсутствуют прикладные файлы/функции, поэтому фиксируются целевые точки, которые должны быть реализованы в коде.

- Определение online/offline:
  - `network.isOnline()`
  - `reportFlow.detectConnectivity()`
- Сбор отчёта:
  - `report.collectReport()`
  - `report.collectTracertTop5()`
- Генерация QR:
  - `qr.buildOfflineMailtoQr()`
- Отправка email:
  - `email.sendViaSmtp()`
  - `email.composeOfflineMailto()`
